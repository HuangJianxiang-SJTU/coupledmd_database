## Supplementary Note 3. Sampling definitions and Gq_8ZPT replica 2

Three sampling quantities must remain distinct:

1. **Nominal protocol sampling:** 207 × 3 × 500 ns = 310.5 µs.
2. **Original file-observed coordinate sampling:** 310.4168 µs.
3. **Reduced-release coordinate coverage:** the represented time between the first and last of 2,500 frames is 499.8 ns per released record.

The 83.2-ns difference between the first two quantities arises from the original `Gq_8ZPT` replica 2 coordinate record, which contains 416.8 ns rather than 500 ns. A continuation-derived coordinate source used for reduced-release repair was independently checked over 10,000 frames. It has monotonic timestamps, no non-finite frame, no invalid box frame and a represented span of 499.95 ns. The audit verdict is `GOOD`.

The repaired source is sufficient for deterministic production of the reduced PDB/XTC record. It is not evidence that a repaired full-system trajectory, topology and preparation input set are deposited. The v12 manuscript reports nominal and original file-observed campaign totals separately.

## Supplementary Note 4. Reduced-record construction

The replica-specific release builder receives a replica number and the frozen 207-system table. Each target system directory contains a copied canonical `structure.pdb` and a generated `traj.xtc`. The PDB defines the retained atoms and ordering.

For a source trajectory already matching the PDB, coordinates are written directly. When a full-system or differently ordered source contains additional atoms, target atoms are mapped by `(resname, resid, atom name)` against a matching source topology. This selection is checked to contain exactly the PDB atom count. The reduction does not infer a ligand-selection rule; users must inspect the released PDB to determine which non-protein components, if any, were retained.

The output schedule is:

| Source form | Selection | Output |
|---|---|---|
| 10,000 or 10,001 frames at ~50 ps | every fourth frame, first 10,000-frame span | 2,500 frames |
| 2,501 frames at ~200 ps | first 2,500 frames | 2,500 frames |
| canonical 2,500-frame reduced source | preserve | 2,500 frames |

All released trajectories use an approximately 200-ps interval and represent 499.8 ns between first and last frame. The reduction excludes membrane lipids, solvent and mobile ions. The underlying full-system trajectories are not included in the reduced release.

## Supplementary Note 5. Full-frame reduced-release QC

The JSON audit for each release replica contains one `system_qc` record per system and an aggregate status. Every frame of each locally complete release is tested for finite coordinates, monotonic finite time and a valid unit cell. The PDB/XTC atom count must match. Reference backbone geometry is used to detect atom-order errors while documented starting-coordinate gaps are excluded from false-positive bond checks. Component geometry is screened for catastrophic chain breaks, coordinate scatter and receptor–G-protein separation.

## Supplementary Note 7. Pocket and gateway records

### Pocket records

Supplementary Data S5 has 2,149 detected-pocket rows from 205 systems. Two additional rows explicitly record valid completed analyses with no persistent pocket: `Gi_7YK6` and `Gi_8YIC`. The system denominator is therefore:

```text
205 systems with detected pockets
+ 2 systems with valid zero-pocket results
= 207 system records
```

The 58-system orthosteric positive-control denominator is defined by the presence of a peptide-based reference and an evaluable pocket analysis. Forty-nine systems recover an orthosteric cluster at the specified 0.85 frequency threshold. Ligand-free systems are not counted as failures. The analysis is a technical check on annotation recovery; pocket recurrence, ranking, biological interpretation and druggability are outside the database paper.

### Gateway records and reproducibility boundary

Supplementary Data S6 contains 5,796 rows:

```text
207 systems × 7 adjacent TM interfaces × 4 metrics
```

The values were calculated from full-system trajectories containing membrane lipids. The reduced release contains no membrane lipids. ## Supplementary Note 8. Worked checksum and loading example

The following complete example reads the replica-1 manifest, locates `G12_7SF7`, verifies both SHA-256 digests and loads the PDB/XTC pair. The user must first extract the relevant family archive without changing member names.

```python
from pathlib import Path
import hashlib
import pandas as pd
import MDAnalysis as mda

release = Path(".")
manifest = pd.read_csv(
    release / "CoupledMD_reduced_trajectory_manifest.csv"
)
row = manifest.loc[
    manifest["system_id"].eq("G12_7SF7")
    & manifest["source_replica"].eq(1)
].iloc[0]

pdb_path = release / row["pdb_relative_path"]
xtc_path = release / row["xtc_relative_path"]

def sha256(path):
    digest = hashlib.sha256()
    with path.open("rb") as handle:
        for block in iter(lambda: handle.read(8 * 1024 * 1024), b""):
            digest.update(block)
    return digest.hexdigest()

assert pdb_path.is_file()
assert xtc_path.is_file()
assert sha256(pdb_path) == row["pdb_sha256"]
assert sha256(xtc_path) == row["xtc_sha256"]

universe = mda.Universe(pdb_path, xtc_path)
assert universe.atoms.n_atoms == int(row["n_atoms"])
assert len(universe.trajectory) == int(row["n_frames"]) == 2500
assert float(row["frame_interval_ps"]) == 200.0
assert float(row["represented_span_ns"]) == 499.8

print(
    row["system_id"],
    universe.atoms.n_atoms,
    len(universe.trajectory),
)
```

The same verification can be performed without Python:

```bash
sha256sum path/from/manifest/structure.pdb
sha256sum path/from/manifest/traj.xtc
```

Both output strings must match the manifest exactly before analysis.
